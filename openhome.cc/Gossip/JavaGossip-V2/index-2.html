物件容器（Container）
    物件容器可以幫您持有對象，在Java中分作兩大類：Collection 與 Map。前者可持有各自獨立的物件，後者持有成對的key-value物件。
  • Collection 類
Collection 類包括了 List 類與 Set 類，List 以放置物件至容器中的順序來排列物件，Set 類不接受重覆的物件，並有自己的一套排序規則。
    • ArrayList
    • LinkedList
    • HashSet
    • TreeSet
    • EnumSet
  • Map 類
在將物件存入 Map類時，需要配合一把key，您要取回物 件時就是根據這把key，Map中的key是唯一的，Map擁有自己的排序機制。
    • HashMap
    • TreeMap
    • EnumMap
輸入輸出（I/O）
    為了要將程式運行結果儲存下來，以供下一次開啟時使用，檔案輸入輸出是一個應用程式所必備的，藉由檔案的輸入輸 出，也可以一同瞭解Java的I/O處理。
  • 檔案處理
File類別是檔 案的抽象表示，也是處理檔案輸入輸出時所 必備的。
    • File
    • RandomAccessFile
  • 位元串流
電腦中的資料都是 以 0 與 1 的方式來儲存，如果您要在兩個裝置之間進行資料的存取，當然也是以 0 與 1 位元的方式來進行，Java將資料目的地與來源之間的流動抽象化為一個串流（Stream），而當中流動的則是位元資料。
    • InputStream、 OutputStream
    • FileInputStream、 FileOutputStream
    • BufferedInputStream、 BufferedOutputStream
    • DataInputStream、 DataOutputStream
    • ObjectInputStream、 ObjectOutputStream
    • SequenceInputStream
    • PrintStream
    • ByteArrayInputStream、 ByteArrayOutputStream
    • PushbackInputStream
  • 字元串流
Reader、 Writer等是處理字元串流 （Character Stream）的相關類別，簡單的說，就是對串流資料以一個字元（16bit）的長度為單位來處理（0~65535、0x0000-0xffff），並進 行適當的字元轉換處理。
    • Reader、 Writer
    • InputStreamReader、 OutputStreamWriter
    • FileReader、 FileWriter
    • BufferedReader、 BufferedWriter
    • PrintWriter
    • CharArrayReader、 CharArrayWriter
    • PushbackReader
執行緒（Thread）
    一個執行緒是進程（Process）中的一個執行流程，一個進程中可以同時包括多個執行緒，CPU會分配時間片段來處理這些子流程，這使得一個程式可以像 是同時間處理多個事務。
  • 執行緒入門
想要實作執行緒功 能，只要實作Runnable介面，單就 表面上來看，執行緒並不困難。
    • 實 作 Runnable 介面
    • Daemon 執行緒
    • 執 行緒生命周期
    • 執 行緒的加入（join）
    • 執 行緒的停止
    • ThreadGroup
  • 同步化（synchronized）
您只要實作 Runnable介面，就可以實作執行緒功能， 入門是簡單的，但資料一被共用，事情就變得複雜。
    • 執 行緒的同步化
    • wait ()、notify()
    • 容 器類的執行緒安全 （Thread-safe）
  • JDK 5.0 新增類別
來看看 JDK 5.0 為多執行緒新增的幾個好用類別。
    • UncaughtExceptionHandler
    • Lock 與 Condition
    • BlockingQueue
    • Callable 與 Future
    • Excutors
反射（Reflection）
    反射機制允許執行時期動態檢視與操作類別，一般應用程式開發較不常用到，但是在一些框架或檢視工具程式中，就經常使用到反射機制。
  • 檢視類別
即使您拿到一個類 別並對它一無所知，但其實它本身就包括了 許多資訊，從 Class 開始，您可以獲得這個類的許多訊息。
    • 簡 介 Class
    • 從 Class 中獲取資訊
    • 簡 介 ClassLoader
    • 自 訂 ClassLoader
  • 生成與操作
使用反射機制，您 可以於執行時期動態載入類別並生成物件， 操作物件上的方法甚至改變類別成員的值。
    • 生 成物件
    • 呼 叫方法
    • 修 改成員值
    • 生 成陣列
metadata
    metadata 簡單的說就是資料的資料（Data about data），J2SE 5.0 中對 metadata 的支援是 Annotation，目的在對程式碼作出說明以利分析工具使用。
  • Annotation
Annotation 對程式運行沒有影響，它的目的在對編譯器或分析工具說明程式的某些資訊，您可以在package、class、method、field等上加上 Annotation。
    • 限 定 Override 父類方法 - Override
    • 標 示方法為 Deprecated - Deprectated
    • 抑制編譯 器警訊 - SuppressWarnings
    • 自 訂 Annotation 型態
  • meta-annotation
在定義 Annotation 型態的時候，為 Annotation 型態加上 Annotation 並不奇怪，這為處理 Annotation 型態的工具提供資訊。
    • 告 知編譯器如何處理 annotaion - Retention
    • 限 定 annotation 使用對象 - Target
    • 要 求為 API 文件的一部份 - Documented
    • 子 類是否繼承父類的 annotation - Inherited
